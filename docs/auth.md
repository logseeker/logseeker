# 認証・ユーザー管理・監査（LogSeeker）

## 方針：任意ON
- 既定は **ログイン不要**（デモ/ローカルはそのまま使える）。
- 別入口の管理パネル（`?screen=administration`、下記「管理パネル」参照）または env `AUTH_REQUIRED=true` で **ログイン必須**に切り替え。
- ONにすると `/api/*` すべてがトークン必須になり、ロールで操作が制御される。OFFの間は誰でも全操作可。

## 管理パネル（通常のログイン後画面とは別入口）
- `ログインを必須にする`ON/OFF・SSO設定・IPアクセス制限は、左メニューの「システム状態」画面には無い。
  `?screen=administration`（左メニューには出さない別URL。通常のログイン画面にも小さくリンクあり）
  からのみ到達する、**専用ログインを持つ別画面**。
- IPアクセス制限は、この管理パネル（このログイン画面自体）へのアクセスを許可IP/CIDRだけに絞る
  もの（SSHの送信元IP制限・WordPress管理画面のIP制限と同じ発想）。ユーザー管理や監査ログなど
  通常のログイン後画面はこの対象外で、今まで通りロール(sysadmin以上)だけで守る（[docs/security.md](security.md)§7）。
- この管理パネルは **管理者(admin)ロールのアカウントのみ**ログインできる（`システム管理者`以下は
  パスワードが合っていてもここでは拒否される。バックエンド `/api/auth/admin-login`）。
- 同じ認証システム・同じユーザーテーブルを使うが、入口（画面・ログインAPI）は分けてある。
  既にadminロールでログイン済み（通常アプリ・別タブ問わず同じブラウザの既存セッション）なら
  再ログインは不要でそのまま入れる。

## 初期管理者アカウント
- ユーザーが1人もいない初回起動時に自動作成：
  - ユーザー名 `logseeker` / パスワード `logseeker`（env `ROOT_USERNAME` / `ROOT_PASSWORD` で変更可）
- **本番では初回ログイン後に必ずパスワードを変更**（ユーザー管理→パスワード再設定）。

## ロール（内部的には Linux の user/sudo/root に相当。画面表示は日本語4段階）
| ロール（表示） | できること |
|---|---|
| **管理者** | 全操作。全ユーザー管理、上位ロールへの昇格、**認証ON/OFF**、**SSO設定**、監査閲覧/DL |
| **システム管理者** | ライセンス変更・通知設定・IOC/API連携・**監査閲覧/DL**・（編集者/閲覧者の）ユーザー作成 |
| **編集者** | 閲覧＋ダウンロード＋**インシデント/コメントの作成・編集** |
| **閲覧者** | 閲覧・ダウンロードのみ |

- sysadmin は viewer/editor のユーザーのみ作成・編集可（sysadmin/adminは触れない）。
- 昇格（sudo/root付与）と認証ON/OFF、SSO設定は admin のみ。
- ロックアウト防止：自分自身の無効化/削除は不可。最後の admin は削除・無効化不可。

## 認証の仕組み（実装）
- パスワードは **pbkdf2_hmac(sha256, 20万回)＋16バイトソルト**（標準ライブラリのみ・依存追加なし）。
- ログインで **Bearer トークン**を発行。DBには**トークンのsha256のみ**保存（平文は保持しない）。既定有効 `SESSION_HOURS=12` 時間。
- フロントは `Authorization: Bearer <token>` を全API呼び出しに付与。401 で自動的にログイン画面へ。

## 監査ログ
- 記録対象：**ログイン/ログアウト**、**変更系API（POST/PUT/PATCH/DELETE）**、**監査/マッピングのダウンロード**、ユーザー作成/更新/削除、ロール変更、認証ON/OFF、SSO設定。
- 閲覧のみ(GET)は記録しない（ノイズ回避）。/ingest（機器→大量）も対象外。
- 「監査ログ」画面で検索・CSVダウンロード可（sysadmin以上）。

## 使い方（ONにする手順）
1. 通常のログイン画面（またはトップページ下部のリンク）から `?screen=administration` を開き、
   管理者アカウント（既定 `logseeker` / `logseeker`）でログイン → パスワード変更。
2. 「ユーザー管理」で必要なユーザーを作成（role を付与）。
3. 管理パネルの「🔐セキュリティ設定」→ **「ログインを必須にする」ON**。
4. 以降、全員ログインが必要。操作は監査ログに残る。

env で最初から必須にする場合は `.env` に `AUTH_REQUIRED=true`（＋ `ROOT_PASSWORD` を強い値に）。
